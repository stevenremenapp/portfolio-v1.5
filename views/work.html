I was the solo frontend developer and initial (not final!) designer working on contract to complete this project with a backend developer and project architect. We were hired as a team to create the client's vision of a "Home Servicing Social Networking Platform" and collaborated with the client to create design documents, meet deliverable deadlines, and shift priorities as needed. This is soon to be in beta release.

Main features include: app access based on authentication, type of user, and subscription (implemented by client request using Stripe Billing, but later removed by client request); private professional profile and project creation with corresponding public pages; project rating; stateful search with filtering; favoriting pros, projects, and photos; connecting consumers with pros via project request messages.

QUICKEN LOANS SINGLE-SIGN-ON VIEWER

TOOLS

  • Angular
  • RxJS
  • AWS DynamoDB
  • AWS Gateway
  • AWS Lambda
  • AWS S3
  • Jasmine

This is a prototype app built during my internship at Quicken Loans to help surface frequently used and difficult-to-access DynamoDB data for engineers working on a Single-Sign-On project. The requirements included using AWS serverless technologies, and I chose Angular because Quicken Loans uses Angular extensively. I worked on this project at the end of my internship, presented it to the engineers using it, iterated on their feedback, and then passed it off before I left.

This was my first project using serverless technologies, and also my first project using RxJS. I learned a lot about different serverless options and how to implement them, albeit on a small scale. I also learned about reactive programming and the Observer pattern as a result of using Angular and RxJS.

TOOLS

I built this project as a learning exercise so I could create and deploy a small full-stack application. It is a platform for collecting and tracking courses used to learn software development, similar to a slimmed-down Goodreads. I built it before starting at Quicken Loans, and because they mentioned using AWS I initially deployed it using AWS EC2 and RDS, but it now lives on Heroku. This was my second project using a Node backend, and my first project in which I wrote my own API endpoints.

I wrote the hide and show shelf animation from scratch, which at the time was an interesting challenge because the animation had to be flexible to handle shelves that were always an unknown height.
